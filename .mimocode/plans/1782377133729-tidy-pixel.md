# Plan: Fix Node.js Process Explosion

## Root Causes Found

### 1. `Particles` component — infinite effect re-runs (HIGH)
**File:** `src/components/animations/particles.tsx:13,99`

The `colors` parameter defaults to `["#f59e0b", "#f97316", "#ef4444", "#a855f7"]` — a **new array on every render**. The useEffect depends on `[count, colors]`, so it re-runs every render, tearing down and recreating the canvas + requestAnimationFrame loop. On the home page (`src/app/page.tsx:31`), `Particles` is used without a `colors` prop, so this default always fires.

**Impact:** Every Framer Motion animation tick on the landing page triggers a re-render → new array reference → effect re-runs → canvas teardown/rebuild → new rAF loop. This cascade is the primary cause of CPU/memory explosion.

**Fix:** Move default colors to a module-level constant so the reference is stable:
```ts
const DEFAULT_COLORS = ["#f59e0b", "#f97316", "#ef4444", "#a855f7"];
```

### 2. `GradientText` — missing `"use client"` (MEDIUM)
**File:** `src/components/GradientText.tsx:1`

Uses React hooks (`useState`, `useRef`, `useCallback`, `useEffect`) and Framer Motion hooks (`useMotionValue`, `useAnimationFrame`, `useTransform`) but has no `"use client"` directive. Turbopack may attempt server-rendering it first, detect hooks, and re-bundle — causing extra compilation passes.

**Fix:** Add `"use client"` as the first line.

### 3. `sharp` — unused native dependency (MEDIUM)
**File:** `package.json:16`

`sharp` is listed in dependencies but never imported anywhere in `src/`. Its native binaries (`@img/sharp-win32-x64`) are present in `node_modules` and Turbopack's module resolver scans them during startup, which on Windows can trigger native module initialization overhead.

**Fix:** Remove `sharp` from `dependencies`.

### 4. `ignoreScripts` + `trustedDependencies` contradiction (LOW)
**File:** `package.json:47-54`

Both `sharp` and `unrs-resolver` are listed in both `ignoreScripts` (blocks scripts) and `trustedDependencies` (allows scripts). In Bun, `ignoreScripts` takes precedence, making `trustedDependencies` dead config. Since `sharp` is being removed, clean up both lists.
